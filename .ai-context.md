## 🛠️ Path: Dev (Core Rules)

*เมื่อระบุว่า "Focus Path: Dev" ให้ AI ปฏิบัติตามกฎเหล่านี้อย่างเคร่งครัด:* [3]

### 1. โครงสร้างและสถาปัตยกรรม (Architecture)

* **Pattern:** ใช้รูปแบบ **Page Object Model (POM)** เพื่อแยกส่วนการระบุตำแหน่งปุ่ม (Locators) ออกจากสคริปต์การทดสอบ [4-6]
* **Folder Structure:**
  * `pages/`: เก็บคลาส Logic หน้าเว็บ (เช่น `login_page.py`, `product_page.py`) [7, 8]
  * `tests/`: เก็บสคริปต์การทดสอบ (Pytest/Playwright) [7, 8]
  * `utils/`: เก็บเครื่องมือเสริม (เช่น `notifications.py` สำหรับ LINE Messaging API) [8]
  * `docs/`: เก็บ Knowledge Base ในรูปแบบ Markdown [9]

### 2. มาตรฐานการเขียนโค้ด (Coding Standards)

* **Language & Runtime:** รองรับ **Python 3.10+** และ **Node.js v22 (LTS)** [5, 10, 11]
* **Package Manager:** ใช้ **pnpm** สำหรับ JavaScript/TypeScript เพื่อความเร็วและประหยัดพื้นที่ [12, 13]
* **Python Quality:** ทุกฟังก์ชันต้องมี **Docstrings** ภายใต้เครื่องหมาย `""" ... """` เพื่อให้ผ่านด่าน Docstring Enforcement Gate [11, 14]
* **Testing:** เน้นการใช้ **Playwright** และรันแบบ **Headless Mode** ใน CI/CD เพื่อประหยัดทรัพยากร [5, 15]

### 3. การจัดการความปลอดภัย (Security & Secrets)

* **Zero-Footprint Standard:** ห้าม Hardcode API Keys หรือ Tokens ลงในโค้ดโดยเด็ดขาด [16-18]
* **Secret Management:** เรียกใช้ค่าผ่านตัวแปรสภาพแวดล้อม (`os.getenv()` หรือ `process.env`) และเก็บค่าจริงใน **GitHub Secrets** หรือ `.env.local` [19-21]
* **Git Integrity:** มั่นใจว่าไฟล์ `.env.local` และ `node_modules/` ถูกระบุไว้ใน **`.gitignore`** เสมอ [22, 23]

## 🚀 CI/CD & Deployment (Optimization)

* **Vercel Quota Guard:** เนื่องจากปัจจุบันติด **Vercel Rate Limited**, เมื่อแก้ไขเฉพาะไฟล์เอกสาร (`.md`) ใน `docs/` ต้องใส่ Tag **`[skip ci]`** ใน Commit Message ทุกครั้ง [24, 25]
* **Workflow Hygiene:** ไฟล์ `.yml` ทั้งหมดต้องจัดเก็บไว้ในพาธ `.github/workflows/` เท่านั้น [11, 26]
* **Conventional Commits:** บังคับใช้ Label นำหน้าข้อความ Commit เสมอ: [27]
  * `feat(dev):` สำหรับฟังก์ชันใหม่
  * `fix(dev):` สำหรับแก้บั๊กหรือ Syntax
  * `docs:` สำหรับงานเอกสาร (ต้องต่อท้ายด้วย `[skip ci]`)
  * `chore:` สำหรับงานจัดระเบียบไฟล์ระบบ

## 📊 Data & Monitoring

* **Database:** ใช้ **MongoDB** (NoSQL) สำหรับเก็บสถิติผลการทดสอบแบบ Batch (insert_many) เพื่อประสิทธิภาพสูงสุด [28, 29]
* **Notification:** ส่งการแจ้งเตือนผ่าน **LINE Messaging API** เมื่อระบบตรวจพบจุดวิกฤต (เช่น ลิงก์สินค้า 404 ติดต่อกัน 3 ครั้ง) [30, 31]
